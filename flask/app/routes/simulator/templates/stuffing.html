Credential theft

[problem statement]

In an attack with stolen credentials the attacker attempts to either gain direct access to an application using the victim’s login and password or to use a known set of credentials across multiple applications to see if the same password has been used anywhere else (credential stuffing).

Even though a simple, non-phishing resistant second factor is enough to protect against these attacks now, they still account for ~30% of all access breaches and especially affect customer-facing applications where MFA suffers from low adoption.

[simulation scenario]

In this scenario you will act as an attacker using stolen credentials to access a legitimate user’s account. The application will give you a set of existing, “stolen” credentials and you will attempt to access the system.

[expected outcome]

Relock passive authentication discovers that you are attempting to access the system from a device that had not been verified and react depending on security policy, thus stopping the attack.
